# AGENTS.md

## 项目协作规范

1. 每次修改必须保证编译完全通过。
2. 单元测试覆盖率超过 95%，并且不要出现无效测试用例。

## 执行要求

- 修改代码前先理解现有 SwiftUI、SwiftData、EventKit 的职责边界，优先沿用项目现有结构。
- 提交前必须运行可用的编译和测试命令；如果本机环境缺少完整 Xcode 等必要条件，必须在交付说明中明确写出阻塞原因。
- 测试必须覆盖真实业务行为，包括任务创建、编辑、删除、LLM JSON 解析、日期解析和 Reminders 同步状态更新；禁止只为覆盖率而写没有断言价值的测试。
- API Key 等敏感信息只能存放在 Keychain，不得写入日志、UserDefaults 或源码。
